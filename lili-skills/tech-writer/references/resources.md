## 文章类型模板

### 技术深度文章
```
标题: [技术]如何[达成某事]:[副标题补充背景]
字数: 2000-4000字
结构:
1. 开篇钩子(当前状况/问题)
2. 核心概念解释(3-5个)
3. 深入分析/实现细节
4. 实际案例/代码示例
5. 优缺点权衡
6. 未来展望
7. 总结和行动建议
```

### 工具/Framework 评测
```
标题: [工具名]深度体验:[一句话评价]
字数: 1500-2500字
结构:
1. 它是什么/解决什么问题
2. 核心功能走查
3. 代码示例(从简单到复杂)
4. 与替代方案对比
5. 适用场景建议
6. 踩坑记录
```

### 趋势分析
```
标题: [领域]的[趋势]:[你的观点]
字数: 1000-2000字
结构:
1. 现象描述(发生了什么)
2. 历史脉络(怎么来的)
3. 关键驱动因素(3-4个)
4. 不同利益相关者的影响
5. 预测和不确定性
```

## 中文技术写作注意事项

### 术语处理
- 首次出现英文术语时附中文解释
- 通用术语用中文(如: 函数、变量、回调)
- 专有名词保留英文(如: React、Kubernetes)

### 代码示例
- 注释用中文
- 变量名保持英文(符合代码规范)
- 输出/日志可适当中文化

### 段落长度
- 中文段落比英文更紧凑
- 多用短句、分段
- 善用列表和表格
